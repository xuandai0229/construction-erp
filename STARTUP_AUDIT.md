# STARTUP_AUDIT.md (Enterprise Production Audit)

## Environment
- Repo: `construction-erp`
- DB: PostgreSQL `construction_erp`, schema `public` @ `localhost:5432`
- App: Next.js (`npm run dev` on `http://localhost:3000`)

## Steps executed
1. `npm install` — success (packages audited; vulnerabilities reported by npm)
2. `npx prisma generate` — success
3. `npx prisma migrate dev --name init` — detected drift; reset prompt appeared.
4. `npx prisma migrate reset --force` — success (DB reset + migrations applied)
5. `npm run dev` — success (Next dev server ready)
6. `npx playwright test enterprise-smoke.spec.ts --reporter=line` — **failed** (auth/database issues)

## Startup / runtime errors observed (NO code fix applied)
### A) Auth session bootstrap fails
**Symptom (server log + test):**
- `POST /api/auth/session` returns **500**
- Browser/test shows: `Development session bootstrap failed: SyntaxError: Failed to execute 'json' on 'Response': Unexpected end of JSON input`

**Root cause (Prisma):**
- `PrismaClientKnownRequestError` P2022
- `Invalid prisma.user.findFirst()` because `User.role` column does not exist in DB

### B) Audit logging table missing in runtime queries
**Symptom:**
- Multiple requests trigger `AuditService.log` and `LoggerService.log`
- Prisma P2021: `The table public.AuditLog does not exist in the current database.`

### C) Notifications relation missing via raw query
**Symptom:**
- `GET /api/workspace/notifications` returns **500**
- Prisma raw query P2010/42P01: `relation "Notification" does not exist`

### D) FiscalPeriod table missing
**Symptom:**
- `prisma.fiscalPeriod.findMany()` fails with P2021: `The table public.FiscalPeriod does not exist in the current database.`

### E) Dashboard API health degraded
- `GET /api/health` returned **503**, causing Playwright test failure

## Reproduction (as observed)
